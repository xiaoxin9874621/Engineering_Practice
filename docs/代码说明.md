# 手写作业OCR识别与自动批改系统 — 代码说明

## 1. 项目概述

本系统是一个面向教育场景的作业批改辅助平台，包含三个核心模块：

- **后端服务（Spring Boot）**：业务逻辑、用户权限、作业管理、批改引擎
- **OCR 微服务（Flask）**：手写文字识别、图像预处理、区域识别
- **Android 客户端（Kotlin）**：教师发布作业、学生拍照上传、批改结果查看

技术栈：Java 17 + Spring Boot 3.2.0 / Python 3.10 + Flask + PaddleOCR / Kotlin + Hilt + Retrofit

### 3.5 服务层（Service）核心逻辑

#### 3.5.1 GradingService — 批改引擎

核心方法 `gradeSubmission(submissionId, ocrTexts: Map<questionNo, ocrText>)`：

```
遍历每道题目，根据 question.type 分三种批改模式：

1. EXACT（精确匹配）—— 选择题、填空题
   - 标准化：trim → 大写 → 去标点
   - ocrText 与 answerKey 完全一致 → isCorrect=1（满分）
   - 不一致 → isCorrect=0（零分）

2. KEYWORD（关键词匹配）—— 简答题
   - answerKey 存储为 JSON 数组，如 ["光合作用","叶绿体","有机物"]
   - 计算命中的关键词比例：hitRatio = matched / total
   - hitRatio ≥ 1.0 → isCorrect=1（满分）
   - hitRatio ≥ 0.5 → isCorrect=2（半对，半分）
   - hitRatio < 0.5 → isCorrect=0（零分）

3. MANUAL（人工批改）—— 默认/未匹配
   - 不自动打分，isCorrect=-1，等待教师手动批改
```

**is_correct 状态码：** 0=错误，1=正确，2=半对，-1=待人工批改

#### 3.5.2 SubmissionService — 提交处理流水线

完整处理流程：

```
1. saveImage()       → 保存图片到 uploads/ 目录
2. asyncProcess()    → @Async 异步执行以下步骤
3. ocrService.recognize(imagePath)
                     → 调用 OCR 微服务识别文字
4. alignOcrTexts()   → 将原始 OCR 文本对齐到题号
5. gradingService.gradeSubmission()
                     → 批改并写入 grading_results 表
```

**OCR 文本对齐逻辑 `alignOcrTexts()`：**

这是系统的关键算法，解决"OCR 识别出的一堆文字如何对应到各题"的问题：

1. **题号检测**：用正则 `^(?:第\s*)?\d+\s*(?:题)?\s*[.．、:：)]?` 匹配题号标记
2. **噪声过滤**：`isNoiseLine()` 去除"数学周测"、"姓名"、"班级"等非答案行
3. **文本提取**：`extractByQuestionMarker()` 从题号标记后提取答案文本
4. **答案提示匹配**：对于无法识别题号的行，与 answerKey 做相似度匹配辅助归类
5. **兜底**：无法归类的文本丢弃

**Mock 模式 `buildDemoOcrTexts()`：**

当 OCR 服务不可用时（如 Apple Silicon 架构限制），直接使用 answerKey 作为 Mock OCR 结果，确保批改流水线可走通。

#### 3.5.3 OcrService — OCR 服务调用

- 使用 `RestTemplate` 调用 Flask OCR 微服务
- 连接超时 3 秒，读取超时 8 秒
- `parseOcrResult()`：解析 JSON 响应，提取 `data.results` 数组
- `isOcrServiceAvailable()`：请求 `/api/health` 健康检查
- 服务不可用时返回空 Map，触发 SubmissionService 的 Mock 降级

## 4. OCR 微服务详解

### 4.1 Flask 路由（app.py）

| 路由 | 方法 | 说明 |
|------|------|------|
| `/api/health` | GET | 健康检查，返回 `{"status": "ok"}` |
| `/api/ocr/recognize` | POST | 图片识别，接收 multipart 图片 + 可选 regions JSON |

识别请求参数：
- `image`：MultipartFile，待识别的作业图片
- `regions`：可选 JSON 字符串，指定识别区域 `[{"x":0,"y":0,"w":100,"h":50}]`

### 4.2 OCR 引擎（ocr_engine.py）

**PaddleOcrEngine — 正式引擎**

- 延迟初始化：首次请求时才加载 PaddleOCR 模型
- 配置：`use_angle_cls=True`（角度分类）、`lang='ch'`（中文）、`use_gpu=False` + `enable_mkldnn=True`（CPU 加速）
- `recognize_by_region(image, regions)`：
  - 有 regions → 对每个区域裁剪后分别识别
  - 无 regions → 全图识别，按 y 坐标排序后自动分配题号

**MockOcrEngine — 降级引擎**

- 当 PaddleOCR 未安装时自动启用
- 返回预设的 Mock 识别结果，用于开发和在 Apple Silicon 上调试

**auto_assign_texts(texts) 算法：**

```
1. 按 y 坐标中心值排序所有识别文本
2. 依次分配题号 1, 2, 3, ...
3. 返回 [{question_no: 1, text: "..."}, ...]
```

### 4.3 图像预处理（image_processor.py）

预处理管线，逐级增强图像质量：

```
原始图像
  → 灰度化（cvtColor COLOR_BGR2GRAY）
  → 高斯模糊（GaussianBlur, kernel=3×3）
  → 自适应二值化（adaptiveThreshold, GAUSSIAN, block=11, C=2）
  → 形态学开运算（morphologyEx MORPH_OPEN, kernel=1×1）
  → 倾斜校正（deskew）
```

**倾斜校正 `deskew()`：**

1. 查找轮廓 → 获取最小外接矩形 `minAreaRect`
2. 计算倾斜角度
3. 角度 > 15° 则跳过（可能是异常轮廓）
4. 角度 ≤ 15° → 仿射变换旋转校正

### 6.2 关键配置说明

- **MySQL 端口 3307**：避免与本地已有 MySQL 3306 冲突
- **OCR 服务可选**：通过 Spring Profile `ocr` 控制，仅在 `docker-compose --profile ocr up` 时启动
- **Apple Silicon 限制**：PaddleOCR 仅提供 amd64 wheel，Rosetta 2 缺少 AVX 指令集会导致 SIGILL 崩溃
  - 解决方案：Apple Silicon 上使用 Mock 模式运行（`ocr.mock-when-unavailable: true`）
  - 或在 amd64 服务器上部署 OCR 服务

### 6.3 启动命令

# 仅启动核心服务（不含 OCR，使用 Mock 模式）
docker-compose up -d

# 查看日志

## 7. 核心流程梳理

### 7.1 学生提交作业 → 批改完成

```
学生拍照/选图
  → Android UploadHomeworkActivity
  → POST /api/submissions/upload (MultipartFile)
  → SubmissionService.saveImage() → 保存到 uploads/
  → SubmissionService.asyncProcess() [@Async]
      → OcrService.recognize() → POST http://localhost:5000/api/ocr/recognize
      → Flask OCR: 图像预处理 → PaddleOCR 识别 → 返回文本列表
      → SubmissionService.alignOcrTexts() → 按题号对齐
      → GradingService.gradeSubmission() → 按题型批改
          → EXACT: 精确匹配 → 0/1
          → KEYWORD: 关键词比例 → 0/1/2
          → MANUAL: 标记待人工批改 → -1
      → 写入 grading_results 表
  → 学生查看批改结果 GET /api/submissions/{id}/grading
```

### 7.2 教师创建作业 → 学生提交

```
教师 CreateAssignmentActivity 填写表单
  → POST /api/assignments (title, classId, questions[])
  → AssignmentService.createAssignment()
      → 计算总分 = Σ question.score
      → 写入 assignments + question_items 表
  → 学生 GET /api/assignments/student → 看到新作业
  → 学生上传图片 → 触发 7.1 的批改流程
```

### 7.3 教师人工批改

```
教师查看提交列表 GET /api/submissions/assignment/{id}
  → 查看批改详情 GET /api/submissions/{id}/grading
  → 对 is_correct=-1 的题目手动评分
  → 添加批注 POST /api/annotations/submission/{id}
      → 保存批注坐标(x,y)、颜色、文字
```

## 9. 设计要点与约束

### 9.1 降级策略

系统设计了多级降级保障可用性：

1. **OCR 服务不可用** → `OcrService` 返回空 Map → `SubmissionService` 使用 `buildDemoOcrTexts()` Mock 数据
2. **PaddleOCR 未安装** → `MockOcrEngine` 自动替代，返回预设结果
3. **Apple Silicon 架构** → Docker Compose 默认不启动 OCR 服务，后端以 Mock 模式运行
4. **Android 无后端** → `HomeworkRepository` Demo 模式，使用本地硬编码数据

### 9.2 批改精度

- **精确匹配**：做了标准化处理（去空格、大写、去标点），但 OCR 识别误差可能导致误判
- **关键词匹配**：支持半对评分（is_correct=2），但依赖 OCR 文本对齐的准确性
- **人工批改兜底**：任何无法自动批改的题目都标记为 -1，等待教师处理

### 9.3 安全设计

- JWT 无状态认证，无服务端 Session
- BCrypt 密码加密
- 接口级角色权限控制（@PreAuthorize）
- 文件上传限制 10MB

### 9.4 异步处理

- 作业提交后的 OCR → 批改流程使用 `@Async` 异步执行
- 用户上传后立即返回，后台处理完成后通过查询接口获取结果
- OCR 服务调用设置超时（连接 3s / 读取 8s），避免阻塞

# 演示流程脚本

本文档用于答辩或项目展示时按顺序演示完整闭环。

## 演示前准备

1. 启动 Docker Desktop。
2. 在项目根目录运行：

```powershell
docker compose up -d --build
```

3. 检查服务状态：

```powershell
docker compose ps
```

4. 打开 Android Studio，运行 Android App 到 Pixel 6 / API 34 模拟器。
5. 准备 `test-data/ocr-images` 下的测试图片，并拖入模拟器相册。

## 演示账号

| 角色 | 用户名 | 密码 |
| --- | --- | --- |
| 教师 | `teacher` | `Demo@123` |
| 学生 | `student` | `Demo@123` |

## 第一步：介绍项目目标

可以这样说明：

```text
本项目解决老师批改手写作业效率低的问题。学生通过手机上传手写作业图片，系统使用 OCR 识别内容，再根据教师设置的题目和标准答案自动批改，最终展示成绩、错题、教师批注和班级统计。
```

## 第二步：教师创建作业

1. 使用教师账号登录。
2. 进入教师端作业页面。
3. 点击创建作业。
4. 创建数学作业，班级 ID 使用 `1`。
5. 添加多道题，例如：

```text
1. 选择题：一次函数图像是什么？答案 A
2. 填空题：植物进行光合作用需要什么？答案 光合作用
3. 简答题：简述一次函数图像特点。关键词 两点、直线、函数
```

6. 提交作业。
7. 返回作业列表，确认新作业出现。

说明点：

- 教师可以创建多道题。
- 作业会绑定班级。
- 后续学生端会根据班级成员关系自动看到该作业。

## 第三步：学生查看待完成作业

1. 退出教师账号，使用学生账号登录。
2. 进入学生端待完成页面。
3. 查看刚才教师发布的作业。

说明点：

```text
学生端调用 /api/assignments/student，后端根据 class_members 表查找学生所在班级，并过滤已经提交过的作业。
```

## 第四步：学生上传手写作业

1. 点击待完成作业。
2. 选择拍照或相册。
3. 从模拟器相册选择测试图片。
4. 确认页面出现图片预览。
5. 点击提交作业。

说明点：

- 图片会上传到后端。
- 后端保存原图。
- 后端调用 OCR 服务。
- OCR 结果会被后端重新对齐，避免标题或题型提示造成答案错位。

## 第五步：学生查看批改详情

1. 提交完成后进入批改详情。
2. 查看上传原图。
3. 查看总分。
4. 查看每道题的 OCR 识别内容、得分、对错和反馈。

如果刚提交时状态还在处理中，可以刷新或重新进入详情。

## 第六步：教师查看提交和添加批注

1. 使用教师账号登录。
2. 切换到教师端提交页面。
3. 查看学生上传上来的作业记录。
4. 点击进入批改详情。
5. 查看学生原图、OCR 结果和每题得分。
6. 在教师批注区域输入批注并保存。

说明点：

- 教师提交页面调用 `/api/submissions/teacher`。
- 只展示教师自己负责班级的学生提交。
- 教师批注会写入 `annotations` 表。

## 第七步：学生查看批注、错题和重新提交

1. 学生进入提交记录。
2. 打开刚才提交的作业详情。
3. 查看教师批注。
4. 进入个人页查看历史成绩和错题。
5. 对同一作业点击重新提交，选择另一张图片再次提交。

说明点：

- 重新提交会覆盖旧图片和旧批改结果。
- 错题列表来自 `/api/submissions/wrong`。
- 待完成数量会在提交后减少。

## 第八步：教师查看统计分析

1. 使用教师账号进入统计分析页面。
2. 查看总体平均分、及格率、批改进度。
3. 查看各班级的提交数、已批改数、平均分和及格率。

说明点：

```text
统计分析调用 /api/statistics/teacher/classes，按教师负责班级聚合数据。
```

## 常见问题处理

### 模拟器访问不了后端

确认 Android 端使用的是：

```text
http://10.0.2.2:8080/
```

真实手机不能用 `10.0.2.2`，需要改成电脑局域网 IP。

### OCR 没结果

检查 OCR 服务：

```powershell
docker compose logs -f ocr-service
```

也可以访问：

```text
http://localhost:5000/api/health
```

### 上传图片失败

检查后端日志：

```powershell
docker compose logs -f backend
```

同时确认图片格式为 JPG/PNG/WebP，大小不超过 10MB。

### 错题列表为空

只有批改完成且存在错误或部分正确题目时，错题列表才会显示内容。

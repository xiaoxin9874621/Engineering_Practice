# 项目总览

## 项目名称

手写作业 OCR 识别与自动批改系统

## 项目定位

本项目演示一个从“教师布置作业”到“学生上传手写作业图片”，再到“OCR 识别、自动批改、教师批注、成绩与错题展示”的完整业务闭环。

适合课程设计、毕业设计、答辩演示、软件工程实践项目展示。

## 使用角色

| 角色 | 主要功能 |
| --- | --- |
| 教师 | 创建作业、设置题目、查看班级提交、查看批改详情、添加批注、查看班级统计 |
| 学生 | 查看待完成作业、上传图片、查看批改结果、重新提交、查看历史成绩和错题 |
| 管理员 | 预留角色，目前主要用于后续扩展 |

## 总体架构

```text
Android App
  -> Spring Boot Backend
  -> MySQL
  -> Redis
  -> Flask OCR Service
  -> PaddleOCR / OpenCV
```

## 技术栈

| 层级 | 技术 |
| --- | --- |
| Android | Kotlin、ViewBinding、MVVM、LiveData、Hilt、Retrofit、Glide |
| 后端 | Spring Boot 3、Spring Security、JWT、MyBatis-Plus、Knife4j / OpenAPI |
| OCR | Python、Flask、PaddleOCR、OpenCV |
| 数据库 | MySQL 8 |
| 缓存 | Redis |
| 部署 | Docker Compose |

## 当前已实现功能

- 登录、注册。
- 教师创建多题作业。
- 教师查看自己创建的作业列表。
- 教师查看自己班级内的学生提交列表。
- 教师查看学生批改详情并添加批注。
- 教师统计分析按班级展示平均分、及格率、提交数和批改进度。
- 学生根据班级成员关系查看待完成作业。
- 学生上传作业图片并预览。
- 同一作业支持重新提交并覆盖旧批改结果。
- OCR 服务 Docker 化。
- OCR 不可用时支持演示兜底。
- OCR 结果按题号、标准答案和关键词重新对齐。
- 每题自动批改。
- 批改详情展示原图、OCR 内容、每题对错、总分和教师批注。
- 学生个人页展示姓名、学号、历史成绩和错题列表。
- Docker Compose 一键启动后端、数据库、Redis 和 OCR 服务。

## 当前限制

- OCR 对潦草手写的准确率受图片质量影响。
- 简答题当前使用关键词匹配，不是大模型语义评分。
- 管理员端功能预留较多，未完整展开。
- 班级基础数据通过初始化 SQL 提供，暂未提供完整班级管理界面。

## 推荐扩展方向

- 增加题目区域框选，提高 OCR 对齐准确率。
- 接入大模型做简答题语义评分。
- 增加 Web 管理端。
- 完善班级创建、学生加入班级、教师分班管理页面。
